# Trabalho-m-dulo-6
Este projeto define a arquitetura lógica e o modelo de dados para um sistema de gestão de casinos. A estrutura foi desenhada para suportar operações de jogo em tempo real, gestão de perfis de utilizadores e controlo de fluxos financeiros.🏗️ Modelagem das EntidadesO sistema está estruturado em torno de dois pilares fundamentais que interagem entre si através de regras de negócio estritas.👤 Entidade: Cliente (User)A entidade Cliente é o centro da operação. Além dos dados demográficos, o sistema gere o estado financeiro e o prestígio (Nível) do utilizador.AtributoDescriçãoObservaçõesIdentificaçãoNome, Nº de ClienteChave única de registo.DemografiaData de Nascimento, Género, NacionalidadeEssencial para conformidade legal (Compliance).ContactoEmail / TelemóvelCanal de comunicação e recuperação de conta.FinanceiroSaldo AtualValor disponível para apostas em tempo real.FidelizaçãoNível (VIP / Standard / etc.)Define benefícios e limites de aposta.EstadoAtivo / InativoControlo de acesso e segurança da conta.🎮 Entidade: Jogo (Game Instance)A entidade Jogo define as regras de funcionamento e os requisitos de entrada para cada modalidade disponível no casino.Categorização Técnica: O sistema distingue jogos por tipo (Com Dealer, Tabuleiros, Peças ou Slots Automáticas).Controlo de Acesso: Cada jogo possui um Nível de Acesso. Um cliente "Standard" pode estar bloqueado de entrar num jogo de nível "High Roller".Mecânica de Custo: Define o Custo Mínimo para participação, garantindo a viabilidade económica da mesa.🔄 Lógica de Interação (Workflow)A relação entre o Cliente e o Jogo não é apenas uma troca de valores, mas um ciclo de eventos validado pelo motor do sistema:Validação de Entrada: O sistema verifica se o cliente está Ativo e se o seu Nível é compatível com o jogo selecionado.Aposta: O cliente retira uma quantia do seu Saldo. O sistema valida se o valor cumpre o Custo Mínimo do jogo.Processamento de Retorno (RNG): O jogo processa a jogada. O retorno financeiro é calculado com base nas probabilidades do jogo, podendo resultar em:Profit (Ganho): Retorno > Gasto.Break-even (Empate): Retorno = Gasto.Loss (Perda): Retorno < Gasto ou Retorno nulo.Atualização de Estado: O saldo do cliente é atualizado instantaneamente e a transação é registada para auditoria.📈 Regras de Negócio e SegurançaRestrição de Idade: O sistema deve validar a Data de Nascimento para impedir o acesso a menores.Gestão de Risco: O Estado do Cliente pode ser alterado para "Inativo" automaticamente caso sejam detetados padrões de comportamento suspeitos ou falta de fundos.Configuração de Retorno: Cada jogo tem um parâmetro de Retorno Financeiro configurável pelo administrador para ajustar a margem da casa (House Edge).
